<h1>Analyzing Protocols with Wireshark</h1>

<h2>Description</h2>
This lab is built to help new users of Wireshark understand the basics of the application, as well as 
expose them to new terminology. Wireshark is a sniffing application, meaning that it has the ability to 
allow users to see what goes on inside their network, otherwise known as traffic, the different pings 
initiated and received by any certain device, as well as packages sent throughout the network. It also 
allows users to create filters to help sift through all the data captured by Wireshark from any given 
number of ports. This is important to Cybersecurity because completed packages sent to and from the 
computer allow users of Wireshark to understand what is going on with the network. Furthermore, 
incomplete packages notify the user of malfunctions or malicious software, making Wireshark a valuable 
tool when upholding network security and assuring the protection of data. The lab also touches on how 
to find time related fields of any given frame, the three different windows within Wireshark, how to 
figure out how many bytes were captured as opposed to how many were sent, as well as how to identify 
the manufacturer of a device based on the first six hexadecimal places of the MAC address. These are all 
valuable tools whether you’re trying to figure out who sent a package, how big the package was, where 
it came from and the IP address associated with it. Altogether, this lab introduced the functionality of 
Wireshark, giving plenty of reasons to believe in its usefulness regarding cybersecurity.
